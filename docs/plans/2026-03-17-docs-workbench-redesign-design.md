# 文档台重设计

日期：2026-03-17

## 背景
当前 `文档台` 同时展开项目、事件、页面、diff 四层信息，但没有清晰主流程。用户进入页面后不知道先看什么，也不知道页面的核心功能是“发现变化”还是“阅读解读”。在事件为空时，页面会呈现大量无效占位，造成“不可用”的感受。

当前问题集中在三点：
- 主任务不清：页面默认没有把“最近哪篇文档变了什么”放到首位。
- 信息层级混乱：项目、事件、页面、解读、diff 都是一级展示，彼此争夺注意力。
- 页面职责混杂：`文档台`、`日报`、`研究台` 对解读深度没有明确分层。

## 目标
把 `文档台` 改成“页面变化发现 + 详细解读 + 深研跳转”的工作台。

目标要求：
- 默认先看最近变更页面，而不是先看事件。
- 页面详情提供“详细解读”，明显强于日报里的简短结论。
- 保留进入 `研究台` 的深度研究入口，并自动携带当前页面上下文。
- `Job` 不再作为文档台主模块，只作为来源信息保留。

## 产品分层
三个页面各做一层，不重复：

- `日报`
  - 简短结论
  - 只回答“今天值得看什么”
- `文档台`
  - 详细解读
  - 解释“这页改了什么、为什么重要、建议继续看什么”
- `研究台`
  - 深度研究
  - 基于当前页面上下文继续做更长报告与证据链

## 信息架构
`文档台` 使用三栏结构，但只保留一个主流程。

### 左栏：项目
只做筛选。

展示：
- 项目名
- 最近变更页数

行为：
- 点击切换项目
- 不放解释文案
- 宽度缩小，避免抢主舞台

### 中栏：页面
这是默认主列表。

排序规则：
- 最近更新优先
- 高重要度页面上浮

每条页面卡只显示：
- 页面名
- 更新时间
- 变更类型：`首读 / 更新 / 删除 / 改写`
- 一句详细标题或短摘要

这里不再先展示“事件列表”。事件降为右侧详情中的关联信息。

### 右栏：解读
选中页面后展示详细解读，固定为以下模块：
- `变化`
- `影响`
- `建议`
- `diff`
- `关联`
- `动作`

各模块职责：
- `变化`：这页改了什么
- `影响`：为什么值得看
- `建议`：接下来应该看什么
- `diff`：新增 / 删除 / 改写摘要
- `关联`：关联事件、来源时间、来源 Job
- `动作`：`深读`、`去研究`

## 交互设计
### 页面卡
用户点击页面卡后：
- 右栏立即更新为该页面的详细解读
- 如果该页面有最近关联事件，则自动显示其标题与摘要
- 如果页面没有事件，也应能直接显示页面 diff 和页面摘要，不出现大片空白

### 深读
`深读` 仍留在 `文档台` 内，不跳页。

展开内容包括：
- 变化背景
- 关键段落
- 可能影响
- 推荐继续阅读页

这是一层“详细增强”，不是完整研究页。

### 去研究
`去研究` 跳到 `研究台`，并带上当前页面上下文：
- 项目
- 页面标题
- 页面摘要
- diff 摘要
- 关联事件标题

这样用户不需要重新组织问题。

### Job 关联
`Job` 不作为 `文档台` 的主模块。

只在 `关联` 区显示一行来源：
- 例如：`来源：2026年3月17日 02:56 / manual-incremental`

点击后跳转到 `线索台` 对应 Job。

原因：
- `Job` 属于生产链路排查
- `文档台` 属于阅读和解读
- 两者要打通，但不能混成一个主界面

## 页面调整
以下内容应删除或降级：
- 顶部 `首次完整解读 / 最近更新 diff` 摘要卡
- 当前把 `事件` 作为一级入口的大区块
- 单独与解读并列的大型 `Page Diff` 区块
- 冗长解释文案和英文装饰性标签

## 失败与空状态
需要重新定义空状态，避免“像坏了”：
- 没有事件但有页面：仍然展示页面摘要与 diff
- 没有 diff：明确提示“这页暂无 diff 记录”
- 没有文档项目：提示“当前没有接入文档项目”
- 读取中：使用简短加载态，不占据大面积说明

## 测试重点
前端测试至少覆盖：
- 页面优先结构替代事件优先结构
- 选择页面后右侧展示 `变化 / 影响 / 建议 / diff`
- 点击 `深读` 展开更多内容
- 点击 `去研究` 触发带上下文的跳转或回调
- 页面没有事件时，右侧仍能展示页面信息和 diff
- 关联区域展示来源时间和 Job 入口
