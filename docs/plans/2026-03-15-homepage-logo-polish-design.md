# 首页品牌头图与 Logo 强化 Design

**Goal:** 在不改变现有功能和路由结构的前提下，把 `架构师开源情报站` 的首页和侧边栏品牌区做成更像正式产品的公开展示版本。

## Context

当前页面已经从后台卡片风格转向情报站封面，但还缺两件关键东西：
- 缺正式品牌识别物，侧边栏只有两行文字，没有徽记和品牌气质
- 首页封面仍然偏“页面标题”，不像一个可对外展示的产品头版

## Options

### 方案 A：只加 Logo
- 在侧边栏补一个图形徽记
- 首页结构不动
- 风险低，但提升有限

### 方案 B：品牌区 + 首页封面联动升级（采用）
- 侧边栏增加徽记、产品说明、导航图标
- 首页封面增加刊头、产品短宣言、结构化信号栏
- 保持原有数据组件不动，只改展示壳层

### 方案 C：整站视觉系统重排
- 连同步监控、情报监控也一起重做
- 效果最好，但当前不值得花这个成本

## Selected Design

采用方案 B。

### 侧边栏
- 增加一个纯 CSS 徽记块，不引入额外图片依赖
- 品牌区显示：`架构师开源情报站` + 一句定位文案
- 导航按钮增加简洁图标前缀，提升扫描效率

### 首页封面
- 保留 `日报首页` 主标题，但把上方刊头换成产品化表达
- 新增产品短宣言：强调“开源动态、中文结论、同步日志”三件事
- 保留现有统计和内容区，避免破坏数据层

### 风格
- 从当前偏冷蓝灰，往“杂志纸张 + 深海蓝 + 黄铜点缀”推进
- 保持浅色背景，增强识别度和产品感

## Testing

- 保留现有 App 测试，并更新对新品牌元素的断言
- 跑 `npm test`
- 跑 `npm run build`
