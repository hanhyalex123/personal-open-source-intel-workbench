# 项目榜设计

**目标**

在封面页新增一个单独的 `项目榜` 区块，只展示 `日报候选项目`，让用户一眼判断“现在该看谁”，而不是继续被静态重要度压住排序。

## 问题

当前首页的核心问题不是卡片不够多，而是排序信号不对：

- 高关注项目即使最近几天没更新，也会继续占首页位置。
- 用户无法快速看出每个项目到底是“刚更新”、“持续在动”，还是“只是历史上重要”。
- 已读行为已经被记录，但首页没有把它变成可见信号。

这导致首页更像静态展示，不像日报首页。

## 设计原则

- 只看 `日报候选项目`，不把全量项目混进首页。
- 先做门槛过滤，再做排序，不让“老而重要”压过“新而有价值”。
- 不上大图表，使用微图表和短标签，保持日报产品的扫读效率。
- 图表只回答一个问题：`这个项目现在值不值得点开`。

## 候选范围

`项目榜` 只取当前日报候选项目：

- `must_watch_projects`
- `emerging_projects`

也就是当前 `/api/dashboard` 里已经进入首页日报的项目集合。

不再额外引入全量项目，也不引入独立筛选模式。

## 准入规则

候选项目先过硬门槛：

- `老牌必看项目`：最近 `must_watch_days` 天内有更新，才允许进入项目榜。
- `近期更新项目`：最近 `emerging_days` 天内有更新，才允许进入项目榜。
- 超出窗口：不进入项目榜，也不因为历史重要度强行保留首页位置。

这条规则和现有日报分区保持一致，避免榜单与首页日报出现两套口径。

## 排序规则

项目榜采用单一排序：`现在该看`。

排序分由 4 个信号构成：

- `新鲜度`：最近一次更新时间，权重最高。
- `重要度`：项目摘要里的 `importance`。
- `7 天更新强度`：最近 7 天内的更新次数与更新分布。
- `已读衰减`：用户最近读过多次的项目适度向后。

推荐默认权重：

- 新鲜度：`0.40`
- 重要度：`0.30`
- 7 天更新强度：`0.20`
- 已读衰减：`0.10`

这里的目标不是做通用推荐系统，而是让首页排序更像日报值班顺序。

## 展示形态

项目榜作为封面页单独区块，位置放在：

- `头条` 下方
- `专题` 上方

区块标题：`项目榜`

每个项目一行，展示 4 组信号：

- 项目名
- 最近更新：`2h / 1d / 5d`
- 7 天趋势：微型 `sparkline`
- 权重分：`82`
- 已读：`3`

辅助手段：

- 重要度继续保留颜色标签，但不再单独主导排序。
- 最近更新时间采用更醒目的短标签，而不是完整日期。
- `sparkline` 使用极简线图或柱点图，不引入完整图表库。

## 交互

- 点击项目榜条目：滚动到对应的日报卡片。
- hover 或点击 `?`：显示更完整的排序拆解：
  - 最近更新时间
  - 最近 7 天更新次数
  - 当前综合分
  - 已读次数

首页默认只显示短信号，不出现长解释文案。

## 后端方案

后端在 `/api/dashboard` 增加一个独立字段，例如：`project_board`。

每个项目对象包含：

- `project_id`
- `project_name`
- `importance`
- `ranking_score`
- `board_score`
- `last_activity_at`
- `last_activity_label`
- `read_count`
- `read_decay_applied`
- `activity_series_7d`
- `updates_7d`
- `llm`

数据来源：

- 候选项目：现有日报 bucket
- 最近更新时间：候选项目 evidence / latest activity
- 7 天趋势：从候选项目对应事件按天聚合
- 已读次数：`read_events`

`board_score` 和现有 `ranking_score` 不必完全相同：

- `ranking_score` 继续服务日报卡片排序
- `board_score` 明确服务项目榜扫读

这样可以避免把首页日报主逻辑和新榜单强耦合。

## 前端方案

新增单独组件，例如：`src/components/ProjectRankBoard.jsx`

职责：

- 渲染榜单行
- 渲染 7 天微图表
- 渲染短标签
- 点击时滚动到对应 `ProjectSummaryCard`

不引入第三方图表库，直接用：

- `SVG polyline` 渲染 sparkline

原因：

- 体积更小
- 可控性更高
- 视觉更贴近当前控制台风格

## 错误与空态

- 如果候选项目为空：显示 `当前没有可上榜项目`。
- 如果 7 天趋势为空：展示 7 个零值点，不显示报错。
- 如果已读记录为空：`已读 0`。
- 如果时间缺失：展示 `暂无更新`，同时把项目排到后面。

## 测试

后端至少覆盖：

- 只从日报候选项目生成榜单
- 超出 3 天/30 天窗口的项目不会进榜
- 最近更新时间更近的项目会排前
- 已读次数高的项目会被轻度降权
- `/api/dashboard` 返回 `project_board`

前端至少覆盖：

- 封面页能渲染 `项目榜`
- 每行能看到 `最近更新 / 权重 / 已读`
- 点击榜单项会滚动到对应日报卡片
- 数据为空时显示空态

## 不做的事

这轮不做：

- 全量项目榜
- 多榜单切换（如“刚更新”“高关注”“最热”）
- 重图表页面或 BI 面板
- 收藏、置顶、用户画像级推荐

先把首页日报候选项目的值班顺序做对。
