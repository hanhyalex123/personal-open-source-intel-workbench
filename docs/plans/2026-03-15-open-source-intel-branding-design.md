# 架构师开源情报站 Design

**Goal:** 把当前项目从通用的“中文运维情报面板”升级为更适合对外展示的产品名“架构师开源情报站”，并补齐 README 的现状说明、架构图和界面截图。

## Scope

- 统一 README、前端侧边栏品牌文案、后端提示词中的产品名称
- 保留现有信息架构与路由，不重命名核心组件文件，避免无收益的大范围重构
- README 增加一张静态架构图和多张当前界面截图，直接反映当前可运行状态

## Approach

### 方案 A：只改 README
- 优点：最快
- 缺点：产品对外名字和应用内名字不一致，截图也会露出旧品牌

### 方案 B：改 README + 应用可见品牌 + 截图同步更新（推荐）
- 优点：对外品牌统一，README 和实际界面一致
- 缺点：需要补一轮截图和少量测试更新

### 方案 C：全量重命名代码结构
- 优点：命名完全一致
- 缺点：组件、脚本、历史文档和测试都会被牵连，收益低，风险高

## Selected Design

采用方案 B。

- 产品名统一为 `架构师开源情报站`
- UI 品牌文案采用两行：`架构师` / `开源情报站`
- 启动脚本和后端提示词同步替换为新名称
- README 采用“现状版”结构：产品简介、能力、界面截图、架构图、启动方式、数据位置、限制
- 图片统一放到 `docs/assets/`，避免污染根目录

## Testing

- 前端：现有 `npm test`
- 后端：`.venv/bin/python -m pytest -q backend/tests/test_start_script.py`
- 构建：`npm run build`
