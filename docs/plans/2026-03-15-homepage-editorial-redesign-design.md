# 首页头版重构设计

## 目标
当前首页存在首屏留白过大、信息块悬空、阅读路径断裂的问题。重构目标是把首页调整为“情报站 / 杂志头版”风格，但同时提高信息密度，让用户进入首页后第一屏就能看到：产品定位、运行快照、主结论入口。

## 设计方向
采用“头版 + 快讯混合型”布局：
- 首屏改成紧凑双栏头版，左侧承载品牌、标题、产品短宣言和主结论入口。
- 右侧整合为运行快照区，汇总最近抓取、最近日报、调度状态、结论总数、项目数。
- 首屏下半部直接露出今日日报首张重点卡，不再要求滚动后才看到核心内容。
- 增量提醒与日报归档改为左右双栏，缩短页面纵向长度。

## 信息架构
### 首屏头版
- 左栏：栏目标签、品牌章、标题、说明文案、值班视角 / 更新节奏 / 产品短宣言。
- 右栏：运行快照卡，承载最近抓取成功、最近日报生成、调度状态、结论总数、项目数。
- 头版下方：今日日报首卡预览区，直接呈现第一条项目摘要。

### 下方内容区
- 左栏：增量提醒
- 右栏：日报归档
- 保留原始数据内容和组件语义，不改接口和数据结构。

## 视觉调整
- 降低大面积空白，压缩 hero 区高度约 25%。
- 用栏线、编号感、情报标签增强头版感。
- 统一首页图标风格，避免零散装饰。
- 统计卡并入头版体系，减少悬空的独立卡组。

## 技术方案
- 仅调整首页组件与样式：
  - `src/components/IntelOverviewPage.jsx`
  - `src/index.css`
- 不变更 API、状态管理或数据结构。
- 优先复用已有 `ProjectSummaryCard`、`IncrementalUpdateList`、`DailyDigestHistory`，只改变布局容器与首页专用样式。

## 测试与验证
- 更新首页相关前端测试断言，确保新文案/结构仍可被稳定定位。
- 运行：
  - `npm test`
  - `npm run build`
