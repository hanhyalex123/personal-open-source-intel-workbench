# Ant Design 控制台与排序体系设计

## 目标

统一前端视觉和交互基线，引入一套明确的 Ant Design 控制台组件体系；同时把“日报排序”和“监控排序”拆成两套不同目标的评分逻辑，避免现在用一套分数同时回答“今天先看什么”和“哪些项目最近持续在动”。

## 已确认规则

- 前端直接引入 `antd`
- 视觉方向统一为控制台风格，不再继续堆自定义散装卡片
- 日报排序与监控排序必须拆开
- 日报更强调“今天先看什么”
- 监控更强调“哪些项目持续变化、值得盯”

## 现状问题

### 1. 前端没有统一组件层

当前页面主要靠：

- `src/index.css`
- 各页面组件中的自定义结构

这导致：

- 相同语义的 UI（卡片、状态、标题、空态、筛选）每页长得不一样
- 页面之间的留白、边框、层级和状态颜色不一致
- 新需求只能继续复制样式，维护成本越来越高

### 2. 封面、线索台、研究台没有统一控制台心智

目前页面更像若干独立专题页，而不是一个统一的工作台：

- 封面：像编辑海报
- 线索台：像临时拼接的状态卡
- 研究台：像表单 + 文本块

用户在不同页面中切换时，操作习惯和阅读顺序都不稳定。

### 3. 排序逻辑目标混杂

当前排序有一个核心问题：同一套分数同时影响日报和监控。

这会带来两个直接副作用：

- 老牌高关注项目即使近期没变化，也容易长期停在上面
- 监控榜会被“日报优先级”带偏，看不出真实变化节奏

## 方案概述

### 1. 引入 Ant Design 5 作为统一组件库

前端统一切到 Ant Design 5，核心原则是：

- 用 `antd` 组件承担通用结构和状态语义
- 只保留业务组件，不再保留大量“视觉即组件”的自定义 CSS
- 用 `ConfigProvider` 统一 theme token

首批统一替换的基础层：

- `Layout`
- `Menu`
- `Card`
- `Statistic`
- `Badge`
- `Tag`
- `Tooltip`
- `Drawer`
- `Tabs`
- `Segmented`
- `List`
- `Table`
- `Spin`
- `Empty`
- `Skeleton`

### 2. 建立“控制台壳层”

顶层应用壳层统一为：

- 左侧导航：`Layout.Sider + Menu`
- 顶部页头：`PageHeader` 风格区域
- 主内容区：统一的 `Content` 容器

公共页面结构统一成：

- 页面标题
- 页面级操作
- 关键状态条
- 主工作区

这样封面、线索台、研究台、设置页的骨架一致，只替换中间业务内容。

### 3. 统一视觉 token

虽然引入 `antd`，但不直接使用默认主题，而是统一控制台 token：

- 主色：冷蓝
- 成功：偏青绿
- 告警：偏橙
- 错误：偏红
- 背景：浅灰白
- 卡片：纯白
- 边框：低对比灰
- 阴影：弱
- 圆角：中小

目标不是做“AntD 默认站”，而是做更像阿里云控制台的业务工作台。

### 4. 页面改造优先级

首批改造 3 页：

1. `封面`
- 统一为控制台式区块
- 强化“头条 / 快讯 / 项目榜 / 专题 / 归档”的阅读顺序

2. `线索台`
- 强化 job 状态、失败来源、日志入口
- 明确“当前运行”和“历史任务”

3. `研究台`
- 统一查询区、证据区、结果区
- 把模型信息、失败原因、证据数量做成标准状态展示

`设置` 页放到第二批，因为它已经刚做过一轮结构改造。

## 排序体系设计

### 1. 日报排序

目标：

- 回答“今天先看什么”

排序因素：

- 新鲜度：45%
- 事件重要度：30%
- 30 天活跃度：15%
- 已读衰减：10%

硬门槛：

- 老牌项目：超过 `must_watch_days` 不进入日报卡片
- 新项目：超过 `emerging_days` 不进入日报卡片

设计意图：

- 首页日报必须更像“值班优先级”
- 今天没变化的项目不该继续占主视觉

### 2. 监控排序

目标：

- 回答“哪些项目最近在持续变化”

排序因素：

- 30 天变化量：35%
- 最近更新时间：30%
- 项目基础权重：25%
- 已读衰减：10%

设计意图：

- 监控榜不是头条榜
- 要突出持续活跃，而不是只看今日优先级

### 3. 可解释性

日报和监控都加统一的排序说明 `?`：

- 统计范围
- 排序因素
- 当前项目为什么排在这里

不在主界面放长说明，只在 tooltip / popover 中解释。

## 测试策略

### 前端

- 组件渲染测试
- 导航/页面骨架切换测试
- 封面、线索台、研究台关键状态展示测试
- `antd` 迁移后的交互回归

### 后端

- 日报排序独立测试
- 监控排序独立测试
- 同项目在两个榜单中的排序差异测试
- 窗口过滤与已读衰减测试

## 非目标

- 本轮不做完整设计系统文档站
- 本轮不重做所有页面
- 本轮不引入 Zustand / Redux 等状态层
- 本轮不做复杂个性化推荐
