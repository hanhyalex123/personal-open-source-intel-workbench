# 封面信息流澄清设计

**目标**

把封面页从“卡片堆叠”收成一条清晰的信息流：先看日报后新增，再看所有日报项目的长期监控，再看今天真正该看的项目卡，最后看历史归档。

## 当前问题

现有封面页的主要问题不是缺数据，而是信息口径不清：

- `项目榜` 只覆盖当前日报候选，不能持续监控用户明确关注的老项目。
- `7天图` 太短，无法看出项目最近一个月到底有没有持续变化。
- `快讯` 只显示增量列表，没有清楚说明它到底回答什么问题，也没有明确点击目标。
- `归档` 只是静态日期卡，不能下钻历史日报。
- `入口` 区块没有信息价值，占用了首屏空间。
- `头条 / 专题` 会继续展示超过窗口且没有更新的项目，导致像 `vLLM` 这样的项目占住主内容位。

## 设计原则

- 默认只展示“现在要判断和要操作的内容”。
- 把长期监控和当天消费分开：监控放 `项目榜`，当天消费放 `头条 / 专题`。
- 所有图表只做信号，不做解释；解释收进 `?`。
- 所有可见列表都要有明确点击后果，不能做死卡片。

## 封面新结构

封面页改成 5 段，从上到下：

1. `头条`
2. `快讯`
3. `项目榜`
4. `专题`
5. `归档`

删除整块 `入口`。

这样首页首先回答：

- 日报生成后有没有新变化
- 哪些项目最近一个月一直在动
- 今天真正需要读哪几张卡
- 历史日报怎么回看

## 快讯定义

`快讯` 表示：**最近一次日报生成之后新增的已分析变化**。

它回答的问题是：`日报发完之后，又发生了什么？`

展示规则：

- 只看 `last_daily_digest_at` 之后的新变化。
- 保留项目级聚合，但每条项目卡要明确显示：
  - 最近变化时间
  - 变化类型
  - 标题
  - 中文结论
- 区块提示文案收成一句：`日报后新增`

点击规则：

- 文档变化：进入 `文档台` 对应项目 / 事件
- release 变化：直接打开来源 URL

## 项目榜定义

`项目榜` 用来监控所有纳入日报体系的项目，而不是只看当天候选。

监控范围：

- `must_watch_project_ids` 中的老项目：始终保留在榜上监控
- `emerging` 近期项目：按当前 `emerging_days` 自动进入榜单

这意味着：

- 老项目即使暂时没更新，也还能在 `项目榜` 里看见它近 30 天的活跃度
- 但它不会再占用 `头条 / 专题`

## 项目榜图表口径

把现有 `7天图` 改成 `30天图`。

统计口径：

- 时间窗：最近 30 天
- 数据源：`release + docs`
- 统计对象：当天新增并完成分析的变化数

默认界面只显示一条微图表，不解释。

`?` tooltip 中解释：

- 最近更新时间
- 30 天总变化数
- release 数
- docs 数
- 已读次数
- 当前排序分
- 排序分构成：新鲜度、重要度、30 天变化强度、已读衰减

## 头条 / 专题规则

`头条 / 专题` 只负责“今天要读的卡片”，不再承担长期监控。

展示规则：

- 老项目：最近 `must_watch_days` 天内有更新，才允许进入
- 新项目：最近 `emerging_days` 天内有更新，才允许进入
- 超出窗口：不显示在 `头条 / 专题`
- 但如果是 `must_watch` 项目，仍保留在 `项目榜`

这会把像 `vLLM` 这种最近没有变化的项目，从封面主卡片区移到监控榜，而不是继续霸占封面。

## 归档定义

`归档` 表示：历史日报入口。

交互改成：

- 日期卡可以点击
- 点击后请求该日期的已归档日报摘要
- 在封面内展开一个 `归档详情` 区块，显示该日期的项目摘要卡列表

不直接重放整页封面，而是优先解决“点不进去”和“无法回看哪天写了什么”的问题。

原因：

- 比整页快照切换更简单可靠
- 直接复用已落库的日报摘要
- 用户更容易理解为“点开这一天看内容”

## 后端调整

需要补两类数据接口：

1. `/api/dashboard`
- `project_board` 改成全量日报项目监控榜
- `activity_series_30d`
- `activity_breakdown_30d`
- `board_explanation`
- `headline_projects` / `topic_projects` 继续只返回窗口内项目

2. 新增历史日报接口
- 例如：`GET /api/daily-digests/<date>`
- 返回该日期的 `summaries`
- 仅使用已归档的日报摘要，不重新生成

## 前端调整

- `IntelOverviewPage` 重排区块顺序
- 删除 `入口`
- `IncrementalUpdateList` 增加点击行为和更短的区块说明
- `ProjectRankBoard` 改成 30 天趋势，并在 `?` 中解释口径
- `DailyDigestHistory` 改成按钮式入口
- 新增 `ArchiveDigestPanel` 或同类组件，承接归档详情

## README 调整

README 需要收成一个开源项目首页，而不是使用说明堆叠。

结构改成：

- 一句话介绍
- 核心能力
- 架构图
- 快速开始
- 配置概览
- 截图

截图需要更新，至少重截：

- 封面
- 线索台
- 文档台 / 设置页（二选一，优先更有代表性的）

## 不做的事

这轮不做：

- 复杂多榜单切换
- BI 级大图表
- 历史日报的整页时间穿梭
- 个性化收藏 / 置顶推荐

先把封面的信息定义和点击路径做对。
